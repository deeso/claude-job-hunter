# Setup — Install Claude Job Hunter Skills

Install all Claude Job Hunter slash commands by symlinking them into your `~/.claude/commands/` directory, and configure a custom working directory for all job hunter outputs (evaluations, cards, profile). Run this once after cloning the repo, or again after pulling updates that add new commands.

## Inputs

**Usage:** `/setup "work-dir"`

Parse `$ARGUMENTS` as an optional quoted string:
- **First quoted string** (optional) — Custom working directory for all job hunter data (profile, evaluations, cards)

If no working directory is specified, the agent will ask the user to choose:
1. Use the repo directory itself (`~/claude-job-hunter/`)
2. Specify a custom directory (e.g., `~/job-search/`, `~/Documents/job-hunt/`)

The working directory is where the user's personal data lives — separate from the repo which contains only the skills and templates.

## Workflow

### Step 1: Locate the repo

Find the `claude-job-hunter` repo by checking (in order):
1. The directory this command file lives in (resolve symlinks to find the repo root — go up one level from `commands/`)
2. `$CLAUDE_JOB_HUNTER_DIR` environment variable
3. `./claude-job-hunter/` relative to cwd
4. `~/claude-job-hunter/`

If not found, ask the user for the path.

### Step 2: Verify repo structure

Confirm the repo has the expected structure:
```
[repo]/commands/*.md          # Slash command files
[repo]/profile/               # Candidate profile template
[repo]/resources/             # Research checklists and references
[repo]/examples/              # Example outputs
```

If anything is missing, warn the user but continue with what exists.

### Step 3: Configure working directory

If a working directory was provided as an argument, use it. Otherwise, ask the user:

```
Where should Claude Job Hunter store your personal data (profile, evaluations, cards)?

1. Use the repo directory: ~/claude-job-hunter/  (simple, everything in one place)
2. Custom directory  (keeps personal data separate from the repo — better for git hygiene)
```

If the user chooses a custom directory:

```bash
mkdir -p [work-dir]/profile
mkdir -p [work-dir]/evaluations
mkdir -p [work-dir]/cards
```

Copy the candidate profile template to the working directory if it doesn't already exist:
```bash
cp [repo]/profile/candidate-profile.md [work-dir]/profile/candidate-profile.md
```

If the profile already exists in the working directory, do NOT overwrite it.

### Step 4: Write configuration

Create a configuration file that all other skills read to find paths:

```bash
# Write to [work-dir]/.claude-job-hunter.conf
```

Contents:
```
# Claude Job Hunter Configuration
# Generated by /setup on [date]
REPO_DIR=[absolute path to repo]
WORK_DIR=[absolute path to working directory]
PROFILE_DIR=[work-dir]/profile
EVALUATIONS_DIR=[work-dir]/evaluations
CARDS_DIR=[work-dir]/cards
RESOURCES_DIR=[repo]/resources
EXAMPLES_DIR=[repo]/examples
```

Also create a symlink or copy the config to a well-known location so skills can find it:
```bash
mkdir -p ~/.claude
ln -sf [work-dir]/.claude-job-hunter.conf ~/.claude/.claude-job-hunter.conf
```

### Step 5: Create commands directory and symlink all commands

```bash
mkdir -p ~/.claude/commands
```

For each `.md` file in `[repo]/commands/` **except `setup.md` itself** (setup is already installed if running this):

```bash
ln -sf [repo]/commands/[command].md ~/.claude/commands/[command].md
```

Commands to install:
- `build-profile.md` — `/build-profile`
- `should-i-apply.md` — `/should-i-apply`
- `mock-interview.md` — `/mock-interview`
- `interview-feedback.md` — `/interview-feedback`
- `comp-research.md` — `/comp-research`
- `offer-negotiation.md` — `/offer-negotiation`
- `player-card.md` — `/player-card`

### Step 6: Verify installation

List all installed symlinks and verify they point to valid files:

```bash
ls -la ~/.claude/commands/
```

For each symlink, check that the target file exists. Report any broken symlinks.

Verify the config file exists and is readable:
```bash
cat ~/.claude/.claude-job-hunter.conf
```

### Step 7: Report results

Present a summary:

```
Claude Job Hunter — Setup Complete

Repo:           [repo path]
Working dir:    [work-dir path]
Profile:        [work-dir]/profile/candidate-profile.md
Evaluations:    [work-dir]/evaluations/
Cards:          [work-dir]/cards/
Config:         ~/.claude/.claude-job-hunter.conf

Installed commands:
  /build-profile        → Build your candidate profile from resume + evidence
  /should-i-apply       → Evaluate whether to apply (scorecard, SWOT, interview prep)
  /mock-interview       → Practice interview loops with feedback
  /interview-feedback   → Debrief after real interviews
  /comp-research        → Market compensation analysis
  /offer-negotiation    → Offer evaluation and negotiation strategy
  /player-card          → Deploy a candidate showcase site

Next steps:
  1. Run /build-profile to create your candidate profile
  2. Run /should-i-apply for your first target role
```

If a `candidate-profile.md` already exists and has content beyond the template, note that it's ready. If it's still the template, remind them to run `/build-profile` first.

---

## How Other Skills Find Paths

All other skills should resolve paths by checking (in order):
1. Read `~/.claude/.claude-job-hunter.conf` for configured paths
2. Fall back to the repo-relative convention (`$CLAUDE_JOB_HUNTER_DIR`, `./claude-job-hunter/`, `~/claude-job-hunter/`)

This means skills work both with and without `/setup` having been run — the config file is preferred but not required.
